一种 基于 DNS 实现 动态 负载 均衡 的 方法 及 DNS 系统 本发明 一种 基于 DNS 实现 动态 负载 均衡 的 方法 ， 采用 Client / Server 架构 ， 其中 ， Server 端 运行 于 DNS 服务器端 ， Client 端 运行 于 业务 服务器 , 所述 Client 端 向 Server 端 发送 业务 服务器 的 状态 信息 , 所述 Server 端 接收 Client 端 发送 的 业务 服务器 状态 信息 ， 并 根据 该 信息 调整 DNS 配置 。 本发明 通过 在 DNS 上 设立 Server 端 ， 在 业务 服务器 上 设立 Client 端 ， Server 端 与 Client 端 通信 连接 后 ， Client 端 将 业务 服务器 的 负载 状况 以及 其他 信息 ， 发送至 Server 端 ， Server 端 通过 都 Client 端 上传 的 信息 进行 分析 ， 对比 于 平均 负载量 ， 增加 或 减少 某一 具体 业务 服务器 的 负载量 。 通过 这一 技术 方案 ， 能够 使 连接 在 DNS 上 的 多台 业务 服务器 的 负载 趋于 平衡 ， 真真 意义 上 实现 了 个 服务器之间 的 负载平衡 。
